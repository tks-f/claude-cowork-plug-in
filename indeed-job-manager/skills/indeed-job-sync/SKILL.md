# Indeed → 求人ボックス・エンゲージ 求人同期スキル

## 概要

Indeedの募集中求人をマスターデータとし、求人ボックス（採用ボード）とengage（エンゲージ）へ同期する。各プラットフォームのUI・SEO特性に合わせて求人内容を最適化して掲載する。

## 同期ルール

| 状態 | アクション |
|------|-----------|
| Indeedにあり、対象サイトにない | 新規作成 |
| Indeedにあり、対象サイトにもある | 内容を比較し差分があれば更新 |
| Indeedになく、対象サイトにある | 対象サイトから削除（非公開） |

## 前提条件

- Claude in Chrome（ブラウザ自動操作）が利用可能
- 以下にログイン済み：
  - Indeed Employer Dashboard（employers.indeed.com）
  - 求人ボックス 採用ボード（saiyo.kyujinbox.com）※ログインは secure.kyujinbox.com
  - engage（en-gage.net/company/）※ログインは en-gage.net/company_login/login/
- 各サイトの日本語UIで操作

## ワークフロー

### Phase 1: Indeedから求人データを収集

1. `employers.indeed.com` にアクセスし求人一覧を表示
2. 「募集中」の求人を全件リストアップ
3. 各求人の詳細ページを開き、以下を収集：
   - 求人タイトル（職種名）
   - 会社名
   - 勤務地
   - 給与
   - 雇用形態
   - 仕事内容
   - 応募資格・条件
   - 勤務時間
   - 休日休暇
   - 福利厚生・待遇
   - アピールポイント
4. 収集データをメモリ上に保持

### Phase 2: 各プラットフォームの既存求人を確認

#### 求人ボックス
1. `saiyo.kyujinbox.com` の採用ボード管理画面にアクセス
2. 「求人一覧」で掲載中の求人を取得（タイトル・ステータス）
3. ステータス種別: 「公開中」「募集停止」

#### エンゲージ
1. `en-gage.net/company/job/` の求人管理画面にアクセス
2. 「求人作成・管理」→「求人管理」から求人一覧を取得
3. ステータス種別: 「公開中」「非公開」
4. 審査状況: 「審査完了」「要確認」

### Phase 3: 差分を判定しユーザーに報告

IndeedデータとPhase 2の既存求人を照合し、以下に分類：

```
■ 同期プラン

【求人ボックス】
  新規追加: 2件
    - システムエンジニア（東京）
    - 営業事務（大阪）
  更新: 1件
    - Webデザイナー（リモート）※給与変更
  削除: 1件
    - カスタマーサポート ※Indeed側で募集終了済み

【エンゲージ】
  新規追加: 2件
  更新: 1件
  削除: 1件

実行してよろしいですか？
```

ユーザー承認を待つ。部分的な実行指示にも対応する。

### Phase 4: 求人内容の最適化

Indeedの求人データを各プラットフォーム向けに変換する。最適化ルールの詳細は `references/platform-optimization.md` を参照。

#### 求人ボックス向け最適化のポイント
- **タイトル**: 「」か/以外の記号を使わない。【】は除外されるリスクあり
- **キーワード**: 求人ボックスのトレンドキーワード（在宅、リモート、未経験歓迎等）を自然に盛り込む
- **雇用形態別分割**: 複数雇用形態は求人票を分けて作成
- **自社評価グラフ**: ラジオボタンで職場特徴をグラフ化（採用ボード独自機能）

#### エンゲージ向け最適化のポイント
- **職種名**: Indeed→Googleしごと検索連携用のシンプルな職種名
- **表示用職種名**: 「年間休日120日以上」「在宅ワークOK」等のアピールキーワード付き
- **仕事内容**: 専門用語を避け具体的に。1日の流れや数字を盛り込む
- **画像・写真**: Indeedに画像があれば活用を提案

### Phase 5: 求人ボックスへの同期実行

#### 新規作成
1. `saiyo.kyujinbox.com` の採用ボード管理画面でヘッダーの「＋求人作成」をクリック
2. 最適化済みの内容をフォームに入力：
   - 募集要項（職種名、雇用形態、給与、勤務地、勤務時間）
   - 仕事の内容
   - 求めている人材
   - 応募・選考について
   - 会社情報
3. 任意項目（職場情報、自社評価グラフ）も可能な限り入力
4. 「公開」をクリック → 審査待ち

#### 更新
1. 求人一覧から対象求人のタイトルをクリックして編集画面を開く
2. 変更箇所のみ更新
3. 「保存」→ 再審査

#### 削除（募集停止）
1. 対象求人のステータスを「募集停止」に変更

### Phase 6: エンゲージへの同期実行

#### 新規作成
1. `en-gage.net/company/job/` で「＋ 新しく求人を作成する」をクリック
   - 又は「コピーして求人を作成する」で既存求人をコピーして編集
2. フォームは14セクション構成で順に入力：
   - 01 雇用形態（必須・ラジオボタン選択: 中途採用(正社員)/中途採用(契約社員)/新卒採用(正社員)/新卒採用(契約社員)/アルバイト・パート採用/業務委託(フリーランス)/インターンシップ/派遣）
   - 02 求人情報（職種名・表示用職種名・仕事内容）
   - 03 会社情報
   - 04 勤務地
   - 05 求人区分
   - 06 給与（最低賃金チェック、固定残業代の記載ルール遵守）
   - 07 勤務時間
   - 08 応募資格・条件
   - 09 試用期間
   - 10 募集人数・募集背景
   - 11 休日休暇
   - 12 待遇・福利厚生
   - 13 選考プロセス
   - 14 画像
3. 「求人掲載にあたっての確認」チェック項目に回答
4. 「入力内容を保存」→「プレビュー」→「掲載方法の選択」で公開

#### 更新
1. `en-gage.net/company/job/` の求人一覧から対象求人の「求人を編集する」をクリック
2. 変更箇所を編集して「入力内容を保存」

#### 削除（非公開）
1. 求人一覧のステータスドロップダウンから「非公開」に変更
2. 又は「求人作成・管理」→「求人削除」メニューから削除

### Phase 7: 完了報告

```
■ 同期完了レポート

【求人ボックス】
  ✓ 新規追加: 2件（審査待ち）
    - システムエンジニア（東京）
    - 営業事務（大阪）
  ✓ 更新: 1件
    - Webデザイナー（リモート）
  ✓ 削除: 1件
    - カスタマーサポート

【エンゲージ】
  ✓ 新規追加: 2件
  ✓ 更新: 1件
  ✓ 削除: 1件

※求人ボックスの新規求人は審査後に掲載されます（通常1〜3日）
```

## UI操作の注意点

- **タブ管理**: Indeed、求人ボックス、エンゲージをそれぞれ別タブで開き切り替える
- **ページ遷移**: 操作後は2〜3秒待ってからスクリーンショットで確認
- **フォーム入力**: テキストエリアは1フィールドずつ入力し、入力後に確認
- **長文入力**: 仕事内容等の長文はペースト操作で入力
- **セッション切れ**: 長時間操作でセッションが切れた場合は再ログインを依頼

## エラーハンドリング

- **ログイン切れ**: ユーザーに再ログインを依頼。パスワード入力はユーザー自身
- **審査落ち（求人ボックス）**: エラー内容をユーザーに報告し、修正案を提示
- **給与エラー（エンゲージ）**: 最低賃金違反等の入力エラーは自動修正を試みる
- **求人マッチング失敗**: タイトルだけで同一求人と判定できない場合はユーザーに確認
