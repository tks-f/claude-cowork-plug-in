# プラットフォーム別 求人最適化リファレンス

Indeed求人データを求人ボックス・エンゲージに変換する際の詳細ルール。

---

## 1. 求人ボックス（採用ボード）最適化ルール

### タイトル規則
- **使用可能記号**: 「」と / のみ。他の記号は除外リスクあり
- **禁止**: 【】＜＞（囲まれた文字ごと除外される）
- **長さ**: 簡潔に。職種名 + 勤務地が基本
- **例**:
  - Indeed: 【急募】システムエンジニア ＜東京勤務＞
  - 求人ボックス: システムエンジニア「東京勤務」

### SEOキーワード戦略
求人ボックスはカカクコムが運営し、「地名×求人」でGoogle/Yahoo!検索上位を獲得。以下のキーワードを自然に盛り込む：

- **勤務形態**: 在宅、リモート、テレワーク、フレックス
- **経験**: 未経験歓迎、経験者優遇、第二新卒
- **待遇**: 土日休み、残業なし、賞与あり、交通費支給
- **雇用**: 正社員、契約社員、アルバイト、パート

### 求人ボックス採用ボード フォーム構成
URL: `saiyo.kyujinbox.com/company/groups/{GROUP_ID}/jobs/edit`

フォームは大きく3セクション構成：

**「仕事について」セクション:**
| フィールド | 必須/任意 | 入力形式 |
|---|---|---|
| 勤務先名 | 必須 | テキスト（会社名を入力） |
| 求人タイトル（職種） | 必須 | テキスト（装飾記号は除去される） |
| 職種区分 | 任意 | ドロップダウン |
| 雇用形態 | 必須 | チェックボックス（正社員/アルバイト・パート/契約社員/業務委託/新卒・インターン/派遣社員/紹介予定派遣） |
| 仕事内容 | 必須 | テキストエリア |
| 求人の特徴 | 任意 | タグ選択（＋特徴を設定する） |
| 写真 | 任意 | 画像アップロード（6枚まで） |

**「働く条件について」セクション:**
| フィールド | 必須/任意 | 入力形式 |
|---|---|---|
| 勤務地 | 必須 | ドロップダウン（都道府県） + テキスト（住所） |
| 交通手段・勤務地補足 | 任意 | テキスト |
| 給与 | 必須 | テキスト |
| 給与補足・福利厚生・受動喫煙防止措置など | 任意 | テキストエリア |
| 勤務時間・休日 | 任意 | テキストエリア |

**「応募・選考について」セクション:**
| フィールド | 必須/任意 | 入力形式 |
|---|---|---|
| 必要項目 | 必須 | ラジオボタン（氏名・連絡先のみ/基本情報/基本情報＋職務経歴） |
| 応募受付の電話番号 | 任意 | テキスト |

右サイドバーに「公開OKまで X/7」の進捗表示あり。「公開する」「一時保存」ボタン。

### 入力フィールドマッピング（Indeed → 求人ボックス）

| Indeedフィールド | 求人ボックスフィールド | 変換ルール |
|---|---|---|
| 求人タイトル | 求人タイトル（職種） | 記号除去、簡潔化 |
| 仕事内容 | 仕事内容 | 段落分け、具体的な業務記述 |
| 応募資格 | 仕事内容に含める | MUST/WANT形式で記述 |
| 給与 | 給与 | 数値明記、月給/年収/時給を明確に |
| 勤務地 | 勤務地 | 都道府県選択＋住所入力 |
| 勤務時間・休日休暇 | 勤務時間・休日 | まとめて記載 |
| 福利厚生 | 給与補足・福利厚生・受動喫煙防止措置など | 箇条書きで網羅 |
| - | 求人の特徴 | Indeedのアピールポイントからタグ選択 |

### 雇用形態マッピング（Indeed → 求人ボックス）
求人ボックスはチェックボックスで複数選択可能：

| Indeed雇用形態 | 求人ボックス雇用形態 |
|---|---|
| 正社員 | 正社員 |
| 契約社員 | 契約社員 |
| アルバイト・パート | アルバイト・パート |
| 業務委託 | 業務委託 |
| 派遣 | 派遣社員 |
| インターン | 新卒・インターン |

### 雇用形態分割ルール
Indeedで複数雇用形態を含む求人は、給与レンジが大きく異なる場合は求人票を分けて作成：
- 正社員（月給25万）とアルバイト（時給1,200円） → 2つの求人票
- 同程度の場合はチェックボックスで複数選択

### 職場情報の生成
Indeedの仕事内容・アピールポイントから以下を生成：
- メンバー構成（例: 20代〜30代が中心の10名チーム）
- 職場の雰囲気（例: 風通しの良い職場で質問しやすい環境）
- 研修・サポート体制

## 2. エンゲージ（engage）最適化ルール

### 職種名の二段構成
engageには「職種名」と「表示用職種名」の2つがある：

- **職種名**: Indeed/Googleしごと検索連携用。シンプルな職種名のみ
  - 例: システムエンジニア
- **表示用職種名**: 採用ページ表示用。アピールキーワード付き
  - 例: システムエンジニア/年間休日120日以上/週2日リモートOK/残業月10h以下

### 表示用職種名に入れるべきキーワード（優先順）
1. 年間休日数（120日以上なら必ず記載）
2. リモート・在宅情報
3. 残業時間（少ない場合）
4. 未経験歓迎（該当する場合）
5. 賞与・昇給情報

### 仕事内容の書き方
engageでは「中学生でも理解できる」レベルの平易さが推奨：

- 専門用語を避け、わかりやすい言葉に置き換え
- 1日の流れを時系列で記載
- 具体的な数字を盛り込む（担当案件数、チーム人数等）
- 箇条書きと文章を混在させて読みやすく

**変換例**:
```
【Indeed原文】
販売管理システムの開発業務。要件定義〜運用保守まで担当。
Java/Spring Bootを使用。

【engage最適化後】
お客様の業務を支える「販売管理システム」の開発をお任せします。

具体的には...
・お客様との打ち合わせで「どんな機能が必要か」をヒアリング
・設計書の作成（先輩がレビューするので安心）
・プログラミング（Java/Spring Bootを使います）
・テスト・動作確認
・リリース後のサポート

1日の流れ（例）：
9:00 朝会でチームの進捗共有
9:30 開発作業
12:00 ランチ（社内カフェあり）
13:00 お客様とオンライン打ち合わせ
14:00 開発作業
17:00 夕会・翌日のタスク整理
18:00 退社（残業は月平均10時間程度）
```

### 入力フィールドマッピング

| Indeedフィールド | engageフィールド | 変換ルール |
|---|---|---|
| 求人タイトル | 職種名 | シンプル化（職種のみ） |
| 求人タイトル + 特徴 | 表示用職種名 | アピールキーワード付加 |
| 仕事内容 | 仕事内容 | 平易化、1日の流れ追加 |
| 応募資格 | 応募資格 | 必須/歓迎を明確に分離 |
| 給与 | 給与 | 最低賃金チェック、固定残業代記載ルール遵守 |
| 勤務地 | 勤務地 | 住所 + 交通アクセス |
| 勤務時間 | 勤務時間 | フレックス等の詳細も記載 |
| 休日休暇 | 休日休暇 | 年間休日数を先頭に |
| 福利厚生 | 福利厚生 | カテゴリ別に整理 |
| アピールポイント | アピールポイント | engage向けに再構成 |

### 給与記載の必須ルール（エンゲージ）
1. 各都道府県の最低賃金を上回ること
2. 手当を含める場合は「一律支給」と明記
3. 一律支給以外の手当は給与に含めない
4. 固定残業代がある場合：時間・金額・超過時の支払いを全て記載
5. 備考欄との整合性を確保

### 雇用形態の選択（最初のステップ）
engageでは求人作成の最初に雇用形態をラジオボタンで選択する。Indeedの雇用形態からマッピング：

| Indeed雇用形態 | engage雇用形態 |
|---|---|
| 正社員 | 中途採用（正社員） |
| 契約社員 | 中途採用（契約社員） |
| アルバイト・パート | アルバイト・パート採用 |
| 業務委託 | 業務委託（フリーランス） |
| 派遣 | 派遣 |
| インターン | インターンシップ |

### フォーム構成（14セクション）
求人作成フォームは以下の順で構成される：
01 雇用形態 → 02 求人情報 → 03 会社情報 → 04 勤務地 → 05 求人区分 → 06 給与 → 07 勤務時間 → 08 応募資格・条件 → 09 試用期間 → 10 募集人数・募集背景 → 11 休日休暇 → 12 待遇・福利厚生 → 13 選考プロセス → 14 画像

### 新フォーマット対応
2025年6月以降、新フォーマットに未対応の求人は外部連携先に連携されない可能性あり。新規作成時は自動的に新フォーマットになる。既存求人は再編集が必要。

## 3. 共通最適化ルール

### 求人マッチング（同一求人の判定）
Indeedとターゲットサイトの求人が同一かどうかを以下の基準で判定：
1. **職種名の類似度**: 記号・装飾を除去して比較
2. **勤務地の一致**: 都道府県 + 市区町村レベル
3. **雇用形態の一致**

3要素中2つ以上一致 → 同一求人と判定。判定が曖昧な場合はユーザーに確認。

### コンテンツ品質チェック
最適化後の求人内容を投稿前に確認：
- [ ] 最低賃金を上回っているか
- [ ] 差別表現がないか（年齢、性別、国籍等）
- [ ] 連絡先情報が正確か
- [ ] 勤務地住所が正しいか
- [ ] 各プラットフォームの文字数制限内か
